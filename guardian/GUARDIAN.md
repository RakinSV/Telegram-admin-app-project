# Guardian Bot — контекст и архитектура

> Читать вместе с CLAUDE.md основного проекта.
> Это ОТДЕЛЬНЫЙ бот от репост-бота, хотя живут в одном Docker-compose.
> Репост-бот — паблишит контент. Guardian — охраняет чат группы.

## Что это

Telegram-бот для защиты чата канала от спама, ботов и токсиков.
Три ключевые функции из задания:
1. Верификация новых участников (капча/вопрос).
2. Предупреждение о правилах при вступлении.
3. Фильтрация спама — два переключаемых режима: стоп-слова vs AI.

Плюс полноценная система модерации поверх этого.

## Архитектура

```
Telegram Group Chat
        ↓ события: new_member, message
Guardian Bot (aiogram 3.x)
        ├── JoinHandler         — верификация новых участников
        ├── SpamFilter          — фильтрация сообщений
        │     ├── режим 1: KeywordFilter (стоп-слова, без LLM)
        │     ├── режим 2: AIFilter (LLM-классификатор)
        │     └── режим 3: HybridFilter (keywords быстро → AI для неясных)
        ├── WarnSystem          — счётчик варнов и эскалация
        ├── AdminCommands       — /warn /mute /ban /trust /stats и т.д.
        └── LogChannel          — пишет все действия в приватный лог-канал
        ↓
SQLite (shared с репост-ботом или отдельный файл)
  таблицы: members, warnings, banned_users, stop_words, trusted_users,
           moderation_log, bot_config, stats_daily
```

## Стек

- `aiogram 3.x` — async Telegram Bot API, FSM из коробки (нужна для
  многошаговой верификации: вопрос → ждём ответа → проверяем).
  Не `python-telegram-bot` — aiogram удобнее для сложных сценариев
  с состояниями (FSM).
- `apscheduler` — кик тех кто не прошёл верификацию за N минут,
  ежедневные отчёты, автосброс варнов по TTL.
- `sqlalchemy` + `alembic` — те же что в репост-боте, общая БД или
  отдельная, через конфиг.
- `openai` SDK — для AI-режима фильтрации (тот же client что в рерайтере).

## Критические технические моменты

### Права бота в группе
Guardian ДОЛЖЕН быть администратором группы со следующими правами:
- Удаление сообщений
- Блокировка пользователей
- Ограничение прав участников (mute = restrict)
- Управление приглашениями (для антирейд-режима)

Без этих прав бот физически не может ничего делать.
Добавить в README: «Добавь бота в группу как администратора».

### Верификация: два подхода, выбрать один на старте

**Подход A — мут до верификации (рекомендуется):**
При вступлении — сразу ограничить права (restrict_chat_member,
can_send_messages=False). Бот пишет вопрос в чат с упоминанием.
Правильный ответ → снять ограничения. Не ответил за N мин → кик.
Минус: новый участник видит чат но не может писать, что может путать.

**Подход B — верификация в личке:**
При вступлении бот пишет в личку. Это чище UX, но требует чтобы
пользователь не закрыл личку с ботом. Многие блокируют ботов.
На старте рекомендую Подход A — надёжнее.

### AI-фильтр: latency и стоимость
Каждое сообщение через LLM — это 100-300ms задержки и ~0.001$ токенов
для gpt-4o-mini. На активном чате (100 сообщ/час) = ~$2.4/день.
Поэтому Гибридный режим — это не просто удобство, это экономия:
AI вызывается только для ~20% сообщений (подозрительных), не для всех.

Промпт для AI-классификатора должен быть коротким и возвращать JSON:
```
{"spam": true/false, "reason": "...", "confidence": 0.0-1.0}
```
Вызывать LLM только если confidence у keyword-фильтра < 0.8.

### Антирейд
Определение рейда: >N новых участников за M минут (по умолчанию >5 за 2 мин).
Действие: вызов setChatPermissions с all=False (Slowmode или полная заморозка).
После успокоения (0 новых за P минут) — автовосстановление прав.
ВАЖНО: сохранять предыдущие разрешения группы перед заморозкой чтобы
восстановить именно их, а не дефолтные.

### Интеграция с репост-ботом
Юзернейм/id репост-бота добавить в таблицу `trusted_users` автоматически
при старте Guardian. Его сообщения НЕ проходят через спам-фильтр.
Это важно: репост-бот публикует контент со ссылками — без whitelist
Guardian будет удалять его посты.

## Структура файлов

```
guardian/
  __init__.py
  config.py               # GuardianSettings (pydantic), отдельно от репост-бота
  bot.py                  # точка входа, инициализация aiogram
  db/
    models.py             # Member, Warning, BannedUser, StopWord, TrustedUser,
                          # ModerationLog, BotConfig, DailyStats
    session.py
  handlers/
    join.py               # верификация новых участников (FSM)
    messages.py           # фильтрация входящих сообщений
    admin.py              # /warn /mute /ban /trust /check /stats /setmode /addword
    reports.py            # /report от участников (P3)
  filters/
    keyword_filter.py     # режим 1: стоп-слова
    ai_filter.py          # режим 2: LLM-классификатор
    hybrid_filter.py      # режим 3: комбинация
    link_filter.py        # отдельный фильтр ссылок (whitelist доменов)
    flood_filter.py       # антифлуд (N сообщ за M сек)
  services/
    warn_system.py        # счётчик варнов, эскалация, TTL-сброс
    captcha.py            # генерация вопросов верификации
    raid_detector.py      # детектор рейда
    log_channel.py        # отправка в лог-канал
    profile_analyzer.py   # анализ профиля нового участника (P2)
  scheduler.py            # APScheduler задачи (кик за тайм-аут, сброс варнов)
```

## Переменные окружения (добавить в .env.example)

```env
# Guardian Bot (ОТДЕЛЬНЫЙ бот от репост-бота)
GUARDIAN_BOT_TOKEN=           # токен Guardian-бота от @BotFather
GUARDIAN_GROUP_ID=            # id группы которую защищаем (отрицательное число)
GUARDIAN_LOG_CHANNEL_ID=      # id приватного канала для логов модерации

# Верификация
CAPTCHA_TIMEOUT_MINUTES=5     # время на прохождение капчи
CAPTCHA_TYPE=math             # math | button | question

# Спам-фильтр
SPAM_MODE=hybrid              # keywords | ai | hybrid
AI_SPAM_CONFIDENCE_THRESHOLD=0.8  # ниже — передаём в AI

# Варны
WARN_THRESHOLD_MUTE=2         # варнов до мута
WARN_THRESHOLD_KICK=3         # варнов до кика
WARN_THRESHOLD_BAN=4          # варнов до перм-бана
WARN_TTL_DAYS=30              # через сколько дней сбрасываются варны
MUTE_DURATION_HOURS=1         # длительность мута по умолчанию

# Антирейд
RAID_JOIN_THRESHOLD=5         # участников за период
RAID_JOIN_WINDOW_MINUTES=2    # период наблюдения
RAID_COOLDOWN_MINUTES=10      # тишина после которой снимается рейд-режим

# Trusted
AUTO_TRUST_AFTER_DAYS=30      # автодоверие через N дней без нарушений
```
